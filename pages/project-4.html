Our First Egg

Godot (LEGACY)

Rock Paper Scissors was made purely for learning Godot. Egg Game was made to add stuff I didn’t/couldn’t to RPS. Dispatch was supposed to be a proper game, but the scope exploded. I took a few weeks off Godot, and now I’m back with a different game. This one is different, because it’s 3D. In the few weeks off, I watched a ton of 3D tutorials and experimented a bit. This game will be nothing big, but finished and polished. I already know blender somewhat, I can make low poly which is good enough for me. During the 3 projects I tried not to use tutorials, but for this one I definitely plan to. Hopefully the scope is small enough to where I can actually finish it. Don’t ask me about the name, I don’t know what February 2026 me was thinking. The absolute core idea and goal is to make something finished. Something that can be played for 5 minutes and then you get an end screen. That's it. Project was started on the 15/02/26.

18/02/26: Commit 7 - Core Rewrite + Bug Fixes

I rewrote a ton in this one. Manager, egg and crate code is now clearer and I fixed a ton of bugs like everything requiring multiple clicks, hover animations being blocked and others. Oh and I also fixed 2 egg bugs: positioning and tweening. In the above footage, you can see some eggs fly from the sky (as intended) and others came up from the ground. That's because I was doing the tween before the scene loaded, since I was instantiating. So it didn't have time to set the starting position but now it's fixed. I duplicate the egg scene instead so it's already there. As for tweening, there were 2 issues. The jittering look was because I was animating a StaticBody3D instead of an AnimatableBody3D. The second issue was that if you click it multiple times, it created multiple tweens. Because the first tween didn’t finish it held up the line for other clicks, which is why everything need multiple clicks. That’s what I think anyways. This was that big rewrite I wanted and I also discovered you can autoload entire scenes which is great. It does sometimes still do the jittering, this is only the case if the player clicks something too quickly after the game starts up. So I might do a small little splash screen or something beforehand.

05/03/26: Commit 11 - Core Gameplay Changes

I am back for Our First Egg v2. Making another game wasn’t exciting so I came back to this because I really enjoyed making it. So I changed the entire gameplay loop and broke everything else. The biggest issue from v1 was that it is repetitive. Click an egg and it always goes to the same crate. But now each crate has an “order”. Each order randomly picks one of the 4 egg types, and is from size 2-5. So each crate no longer only accepts 1 type of egg, instead you fill up the order. I think this is a lot better, I plan to add a lot more egg types to make it more interesting. But now that the loop has changed, the upgrades are not relevant and a lot of the code is commented out so I’ll have to clean that up. I’m exciting to make v2 :D

v2 Review

I think v2 turned out great. Towards the end when it become mainly polish and bug fixes, it wasn't as fun because I had to redo or remove a ton of stuff. However in terms of gameplay, this is a lot better than v1. This is what my next game would've been, except I would've had some AI in there but I figured I can just update this one instead of essentially remaking the entire game. Biggest issue I had with v2 are race conditions. So something might happen before something else and it either looked bad or it caused a bug. It's what happened with the crate, a new order was created before it's sell animation could finish locking it into the idle state forever. So hopefully the next game is clearer. My favourite part of v2 has to be the crates, they levelled up so much and the code is also a lot clearer. I do like the upgrades though they do just come down to making the number bigger. Overall, I think the game is complete after v2 but only in terms of gameplay, I would've preferred to do more polish in some areas especially when it comes to interactions.

Project Review

Overall, this game was a lot of fun to make. I didn't experience any major bugs, except the tweening bugs in v1 and the race conditions in v2. My editor was also being really weird during v2 so that was annoying. There are a couple of things I would've like to add but didn't. First is save persistence, I don't think it was necessary because the game is only like 5 minutes. Second thing is I should've done a lot more testing. It's technically possible to softlock the game because if you bin all the eggs for example, then the game doesn't know that so you can't progress. There's some minor polish stuff I would've like to add but I think it's best to leave it where it is for now. If I do come back for v3, it'll be very minor stuff and architectural stuff changes, not the core gameplay. I’ll keep a list of some changes I’ll make later on. I don’t think I’ll make anything for a few weeks now.
